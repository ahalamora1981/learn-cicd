## 3. 编写CI/CD配置文件

### 步骤3.1：创建GitHub Actions目录

```powershell
mkdir -p .github/workflows
```

### 步骤3.2：创建基础CI配置文件

创建文件 `.github/workflows/ci.yml`，内容如下：

```yaml
name: CI

on:
  push:
    branches: [ main, master ]
  pull_request:
    branches: [ main, master ]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v4
    
    - name: Set up Python
      uses: actions/setup-python@v5
      with:
        python-version: '3.12'
    
    - name: Install uv
      run: |
        curl -LsSf https://astral.sh/uv/install.sh | sh
        echo "$HOME/.local/bin" >> $GITHUB_PATH
    
    - name: Install dependencies
      run: |
        uv sync
    
    - name: Run tests
      run: |
        uv run pytest -v
```

这个配置文件的作用：
- **触发条件**：当push到main/master分支，或者创建PR时触发
- **运行环境**：Ubuntu最新版
- **执行步骤**：
  1. 检出代码
  2. 安装Python 3.12
  3. 安装uv
  4. 安装项目依赖
  5. 运行测试

### 步骤3.3：创建代码质量检查配置

创建文件 `.github/workflows/lint.yml`，内容如下：

```yaml
name: Lint

on:
  push:
    branches: [ main, master ]
  pull_request:
    branches: [ main, master ]

jobs:
  lint:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v4
    
    - name: Set up Python
      uses: actions/setup-python@v5
      with:
        python-version: '3.12'
    
    - name: Install uv
      run: |
        curl -LsSf https://astral.sh/uv/install.sh | sh
        echo "$HOME/.local/bin" >> $GITHUB_PATH
    
    - name: Install dependencies
      run: |
        uv sync
        uv add ruff
    
    - name: Run ruff check
      run: |
        uv run ruff check .
    
    - name: Run ruff format check
      run: |
        uv run ruff format --check .
```

这个配置文件的作用：
- 使用ruff进行代码风格检查
- 检查代码格式是否符合规范

### 步骤3.4：提交配置文件

```powershell
git add .github/
git commit -m "ci: 添加GitHub Actions CI配置"
git push
```

## 4. 测试CI/CD流程

### 步骤4.1：查看CI运行状态

推送后，打开你的GitHub仓库页面，点击 "Actions" 标签页，你会看到CI正在运行。

或者用命令行查看：

```powershell
gh run list
```

### 步骤4.2：查看CI运行详情

```powershell
# 查看最新一次运行的详情
gh run view --log

# 或者查看特定运行的详情
gh run view <run-id>
```

### 步骤4.3：测试PR触发CI

我们来创建一个分支，修改代码，然后创建PR，测试PR是否能触发CI。

```powershell
# 创建新分支
git checkout -b feature/add-health-check

# 修改main.py，添加一个health check端点
```

在 `main.py` 中添加以下代码：

```python
@app.get("/health")
def health_check():
    return {"status": "healthy"}
```

然后在 `test_main.py` 中添加测试：

```python
def test_health_check():
    response = client.get("/health")
    assert response.status_code == 200
    assert response.json() == {"status": "healthy"}
```

```powershell
# 提交并推送
git add .
git commit -m "feat: 添加健康检查端点"
git push -u origin feature/add-health-check

# 创建PR
gh pr create --title "添加健康检查端点" --body "添加了/health端点，用于检查服务是否正常运行"
```

创建PR后，CI会自动运行。你可以在GitHub上查看PR的CI状态。

### 步骤4.4：合并PR

如果CI通过，你可以合并PR：

```powershell
# 合并PR
gh pr merge --merge

# 切换回main分支
git checkout main
git pull
```

## 5. 进阶：自动部署

### 步骤5.1：创建部署配置

创建文件 `.github/workflows/deploy.yml`，内容如下：

```yaml
name: Deploy

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    needs: test  # 等待CI测试通过后才执行
    
    steps:
    - uses: actions/checkout@v4
    
    - name: Set up Python
      uses: actions/setup-python@v5
      with:
        python-version: '3.12'
    
    - name: Install uv
      run: |
        curl -LsSf https://astral.sh/uv/install.sh | sh
        echo "$HOME/.local/bin" >> $GITHUB_PATH
    
    - name: Install dependencies
      run: |
        uv sync --only-group main
    
    - name: Deploy to server
      env:
        DEPLOY_KEY: ${{ secrets.DEPLOY_KEY }}
        SERVER_HOST: ${{ secrets.SERVER_HOST }}
        SERVER_USER: ${{ secrets.SERVER_USER }}
      run: |
        echo "Deploying to server..."
        # 这里是示例，实际部署命令需要根据你的服务器配置
        # 例如：
        # ssh -i $DEPLOY_KEY $SERVER_USER@$SERVER_HOST "cd /app && git pull && uv sync && systemctl restart myapp"
        echo "Deployment completed!"
```

**注意**：这个部署配置需要你配置GitHub Secrets。

### 步骤5.2：配置GitHub Secrets

```powershell
# 设置secrets（需要在GitHub网页上操作，或者用以下命令）
# 注意：这些命令需要你有相应的权限

# 查看当前的secrets
gh secret list

# 添加secret（示例）
# gh secret set DEPLOY_KEY
# gh secret set SERVER_HOST
# gh secret set SERVER_USER
```

或者在GitHub网页上：
1. 进入你的仓库
2. 点击 "Settings"
3. 点击 "Secrets and variables"
4. 点击 "Actions"
5. 点击 "New repository secret"
6. 添加以下secrets：
   - `DEPLOY_KEY`: 你的SSH私钥
   - `SERVER_HOST`: 服务器地址
   - `SERVER_USER`: 服务器用户名

### 步骤5.3：提交部署配置

```powershell
git add .github/workflows/deploy.yml
git commit -m "ci: 添加自动部署配置"
git push
```

## 6. 常见问题

### Q1: CI运行失败怎么办？

```powershell
# 查看CI运行日志
gh run view --log

# 重新运行失败的CI
gh run rerun <run-id> --failed
```

### Q2: 如何手动触发CI？

在配置文件中添加 `workflow_dispatch` 触发器：

```yaml
on:
  push:
    branches: [ main, master ]
  pull_request:
    branches: [ main, master ]
  workflow_dispatch:  # 添加这一行，允许手动触发
```

### Q3: 如何查看CI运行历史？

```powershell
# 查看所有运行记录
gh run list

# 查看特定工作流的运行记录
gh run list --workflow=ci.yml
```

### Q4: 如何取消正在运行的CI？

```powershell
# 取消最新一次运行
gh run cancel

# 取消特定运行
gh run cancel <run-id>
```

### Q5: 如何在CI中使用环境变量？

在配置文件中添加：

```yaml
env:
  MY_VAR: my-value
```

或者使用GitHub Secrets：

```yaml
env:
  MY_SECRET: ${{ secrets.MY_SECRET }}
```
